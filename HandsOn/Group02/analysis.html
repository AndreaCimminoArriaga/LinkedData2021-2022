License of the new dataset

Following with the requisites that are stated on the license of the Madrid's City Hall, we can put any license as long as:

  • It is prohibited to desnaturilize the sense of the information
  • It should be cited where the dataset was extracted from as "Origin of the data: Madrid's City Hall"
  • Include last modification date
  • It may not be indicated, implied or suggested that the Madrid City Council participates, sponsors or supports the reuse that is carried out with the information.

So we decided to choose a Copyleft License including all the previous mentioned bullet points. By the very nature of the license, people that use our data must provide it's application as an open resource

In conlusion, the user of our dataset has the same liberties as we have when using the Madrid's City Hall dataset.

Resource naming strategy

  • Uses a domain that is not the one given by default in Protégé
  • Uses different paths for ontology resources (i.e., classes and properties) and individuals
  • Ensures that the paths for individuals of different classes are not the same
  • Defines individual URIs independently of class URIs

We have used the following name strategies:

  • For our ontology: http://www.consumo_madrid.com/ontology#
  • For our data: http://www.consumo_madrid.com/resource/

We have defined four classes:

  • A main class "Building", from which the other classes will inherit
  • "Neighborhood"
  • "District"
  • "Energy_use"

The class "Building" has object properties such as "hasEnergyUse", "isFromDistrict" and "isFromNeighborhood" which connects with the classes "Energy_use", "District", "Neighborhood" respectively and "isLocatedAt" which references to an rr:IRI

And data properties such as:

  • "hasId": xsd:int
  • "latitude": xsd:float
  • "longitude": xsd:float
  • "buildingName": xsd:string
  • "hasNumber": xsd:int
  • "hasPostalCode": xsd:int
  • "buildingType": xsd:string

The class "Neighborhood" has the object property "sameAs" and the following data properties:

  • "label": xsd:string
